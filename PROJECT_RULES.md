# Folder Rules

1. All analysis work in this folder must be delivered as a beginner-level `.ipynb` notebook.
2. Keep the full analysis code inside the notebook. Do not move logic to separate script files unless the user explicitly asks for that.
3. Prefer the easiest clear code over compact or clever code.
4. Use short Markdown sections so a beginner can follow the workflow step by step.
5. Add enough comments in code cells to explain what each block is doing.
6. Use plain variable names that match the measurement meaning.
7. Put user-tunable settings in one small parameter cell near the top.
8. Make notebooks easy for manual check:
   - show summary tables
   - show a few rows of marker tables
   - make plotting labels explicit
   - save the final figure to a PNG file
9. When a value is skipped by a rule, show that clearly in the notebook summary.
10. Default output layout should be simple and easy to inspect by eye.
